## 6. superap 캠페인 매체 타겟팅 버그 수정

### 문제
- superap.io에서 캠페인 수정 페이지 확인 시 **매체 타겟팅이 전부 꺼져 있음**
- 캠페인 등록/수정 시 설정한 이미지도 반영되지 않음
- 최근 캠페인들의 집행이 제대로 안 되고 있었음

### 원인 분석
- `_build_media_fields()`가 `targetMediaIds`를 `files=` 파라미터로 전송
  ```python
  # 수정 전 (문제 코드)
  return [("targetMediaIds", (None, str(mid))) for mid in media_ids]
  # files= 에 추가되어 multipart file-like 필드로 전송됨
  ```
- `_build_image_fields()`도 마찬가지로 `image_url`을 `files=`로 전송
- superap 서버는 HTML 표준 폼 제출 방식을 사용하여, multipart file 필드에서 이 값들을 파싱하지 못함

### 수정 내용
- `targetMediaIds`와 `image_url`을 `data=` 파라미터로 분리하여 일반 폼 데이터로 전송

```python
# 수정 후
def _build_image_fields(self):
    return [("image_url", icon), ("image_url", img1), ("image_url", img2)]

def _build_media_fields(self):
    return [("targetMediaIds", str(mid)) for mid in media_ids]

# 3개 메서드 모두 동일하게 수정
files = [(k, (None, v)) for k, v in form_data.items()]
extra_data = self._build_image_fields() + self._build_media_fields()
resp = self.session.post(url, data=extra_data, files=files, ...)
```

### 적용 범위
- `register_campaign` (신규 캠페인 등록)
- `extend_campaign` (캠페인 연장/수정)
- `update_campaign` (캠페인 예산/정답 변경)
- Active 캠페인 25개 전체에 매체 타겟팅 + 이미지 일괄 재적용 완료

### 주요 파일
- `services/superap_client.py` - `_build_image_fields()`, `_build_media_fields()`, 3개 POST 메서드

## 7. superap 캠페인 자동 관리 시스템

### 스케줄러 작업 (3분 간격)
1. **캠페인 자동 세팅** (`campaign_scheduler.py`): 대기 주문 → 캠페인 등록/연장 → in_progress
2. **캠페인 완료 체크** (`campaign_scheduler.py`): in_progress → TotalOff 확인 → completed
3. **리메인 동기화** (`campaign_scheduler.py`): superap action_count → 인스타몬스터 remains 업데이트
4. **유튜브 캠페인** 동일 3가지 (유튜브 구독자용)

### 캠페인 설정 파일
- `campaign_settings.json` - 인스타그램 캠페인 설정 (이미지 URL, 매체 타겟팅 ID 등)
- `campaign_map.json` - username → ad_idx 매핑

## 9. 캠페인 TotalOff 시 자동 부분환불 구현

### 작업 내용
- superap 캠페인이 TotalOff(종료)되었을 때 `action_count < quantity`인 경우 자동 부분환불
- 인스타몬스터에서 남은 수량을 취소하여 잔액 환불
- Cafe24는 건드리지 않음 (InstaMonster + ADBC만 처리)
- 별도 텔레그램 봇으로 환불 알림 전송

### 처리 흐름
```
TotalOff 감지 (3분마다)
  ├─ action_count >= quantity → completed (기존 동작)
  └─ action_count < quantity (부분이행)
       ├─ 1. remains 동기화 (quantity - action_count)
       ├─ 2. 인몬 주문 취소 → 자동 환불
       ├─ 3. ProcessedOrder.status = "partial_refund"
       └─ 4. 별도 텔레그램 봇으로 알림
```

### 안전장치
- remains 동기화 실패 시 취소 진행하지 않음 (다음 3분 사이클에서 재시도)
- 인스타/유튜브 캠페인 모두 적용

### 예산 재계산 방식 (새 주문 시)
- `new_budget = action_count + (새 수량 × 1.2)`
- action_count 기준이므로 부분환불 후 새 주문 100명이면 정확히 100명분만 추가 작업

### 수정 파일
- `config.py` - `TELEGRAM_REFUND_BOT_TOKEN`, `TELEGRAM_REFUND_CHAT_ID` 추가
- `services/telegram_notifier.py` - `_send_refund()`, `notify_partial_refund()` 추가
- `services/campaign_scheduler.py` - `_process_partial_refund()` 헬퍼 + 완료체크 분기 로직
- `models.py` - `partial_refund` 상태 추가 (주석만, 스키마 변경 없음)
